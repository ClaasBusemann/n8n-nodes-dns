Implement the DNS Watch polling trigger node in `src/nodes/DnsWatch/DnsWatch.node.ts`.

## Context

From the design document, section **DNS Watch Node (Trigger)**:

> A polling trigger that checks a DNS record at a configurable interval and fires the workflow when the answer changes.

Additional parameters beyond the Lookup node:

| Parameter | Type | Description |
|-----------|------|-------------|
| Poll Interval | Number | Polling frequency in minutes. Minimum: 1. Default: 5 |
| Fire On | Options | "Any Change", "Record Appears", "Record Disappears", "Value Matches" |
| Expected Value | String | For "Value Matches" mode |

**State management:**

> The trigger persists the previous answer set using `this.getWorkflowStaticData('node')`. On each poll it compares the new answer to the stored state and fires only when the configured condition is met.

From section **File Responsibilities**:

> **`DnsWatch.node.ts`** — The polling trigger node class. Implements `poll()` which calls the same DNS client as the action node, compares results against state persisted via `this.getWorkflowStaticData('node')`, and returns items only when the configured condition is met. Contains no packet encoding, no parsing, no UDP logic.

## Tasks

- Implement the `DnsWatch` polling trigger node class
- Share all DNS query parameters with the Lookup node
- Add trigger-specific parameters: Poll Interval, Fire On, Expected Value
- Implement `poll()` method that:
  1. Queries DNS using the same client as the Lookup node
  2. Compares new answer to state persisted via `this.getWorkflowStaticData('node')`
  3. Returns items only when the configured fire condition is met
  4. Updates persisted state
- Implement all four Fire On modes: Any Change, Record Appears, Record Disappears, Value Matches
- Node contains no packet encoding, no parsing, no UDP logic

## Acceptance Criteria

- Trigger fires correctly for each of the four Fire On modes
- State is persisted between polls via `getWorkflowStaticData('node')`
- Identical consecutive responses do not trigger the workflow
- Node reuses the DNS client from the Lookup node
