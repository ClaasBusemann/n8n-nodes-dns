Scaffold the n8n community node project and set up all development tooling.

## Context

From the design document, section **Scaffolding and Tooling**:

> The project will be scaffolded using the `@n8n/node-cli`: `npm create @n8n/node`
> This generates the correct project structure, TypeScript configuration, ESLint rules (including `eslint-plugin-n8n-nodes-base`), and build tooling.

## Tasks

- Run `npm create @n8n/node` to generate the project skeleton
- Verify the generated `tsconfig.json`, ESLint config, and project structure match the **Package Layout** section
- Create the `justfile` with all recipes defined in the design document (build, dev, lint, lint-fix, format, format-check, test-unit, test-int, test, test-coverage, test-e2e, test-all, verify, check)
- Create `verify.mjs` as a local replica of `@n8n/scan-community-package`
- Configure Jest with `ts-jest` for the three test layers (unit, integration, e2e)
- Configure Jest coverage thresholds per the **Test Infrastructure** table: 90%+ line coverage on `transport/` and `utils/`, 80%+ overall
- Ensure the `src/` directory structure matches the package layout: `nodes/DnsLookup/`, `nodes/DnsWatch/`, `credentials/`, `transport/`, `utils/`
- Ensure the `test/` directory structure has `unit/`, `integration/`, and `e2e/` subdirectories

## Acceptance Criteria

- `just build` compiles successfully
- `just lint` runs without configuration errors
- `just test-unit` runs (even with zero tests)
- `just test-coverage` enforces configured coverage thresholds
- Project structure matches the **Package Layout** section of the design document
