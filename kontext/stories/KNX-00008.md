Implement SPF and DMARC TXT record parsers in `src/utils/txt-parsers.ts`.

## Context

From the design document, section **Output Format > TXT Record Parsing**:

**SPF** â€” detected by the `v=spf1` prefix. Mechanisms are parsed into an array preserving order and qualifier (`+` pass, `-` fail, `~` softfail, `?` neutral). The default qualifier is `+` when omitted.

**DMARC** â€” detected by the `v=DMARC1` prefix. Tags are parsed into named fields. Missing optional tags are populated with their RFC 7489 defaults (e.g. `adkim` and `aspf` default to `relaxed`, `pct` defaults to 100).

From section **File Responsibilities**:

> **`txt-parsers.ts`** â€” Pure functions. Chain of detection functions that test raw TXT values against known formats and return structured parsed objects. Short-circuits on first match.

> Malformed records (correct prefix but invalid body) return `parsed: null` with a `parseError` string field rather than crashing.

## Tasks

- Implement SPF parser: detect `v=spf1` prefix, parse mechanisms with qualifiers (`+`, `-`, `~`, `?`), types (`ip4`, `ip6`, `include`, `a`, `mx`, `all`, `redirect`, `exists`), and values
- Implement DMARC parser: detect `v=DMARC1` prefix, parse all tags (`p`, `sp`, `rua`, `ruf`, `adkim`, `aspf`, `pct`, `fo`, `rf`, `ri`), fill RFC 7489 defaults for missing optional tags
- Implement the detection chain pattern: each parser tests the raw value and returns a parsed object or `null`, short-circuiting on first match
- Handle malformed records: return `parsed: null` with `parseError` field
- Write unit tests (TDD) per the **Testing Strategy > Unit Tests > txt-parsers.ts** section:
  - SPF: parse mechanisms with all qualifier types, `ip4`, `ip6`, `include`, `a`, `mx`, `all`, `redirect`, `exists`
  - SPF: default qualifier is `+` when omitted
  - SPF: multiple mechanisms in a single record
  - DMARC: parse all tags
  - DMARC: missing optional tags filled with RFC 7489 defaults
  - DMARC: multiple `rua`/`ruf` URIs separated by commas
  - Malformed SPF/DMARC (correct prefix, invalid body): return `parsed: null` with `parseError`

## Acceptance Criteria

- SPF output matches the example structure in the design document
- DMARC output matches the example structure in the design document
- RFC 7489 defaults are correctly applied for missing DMARC tags
- Malformed records produce `parseError`, not crashes
- All unit tests pass
