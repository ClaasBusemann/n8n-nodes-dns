Implement the DNS name compression codec in `src/utils/name-compression.ts`.

## Context

From the design document, section **Transport Layer > Wire Format**:

> **Name compression** — response packets may contain pointers (two bytes with the high two bits set to `11`) that reference earlier positions in the packet. The decompressor follows pointer chains with a recursion limit to prevent malicious loops.

From section **File Responsibilities**:

> **`name-compression.ts`** — Pure function. Decompresses DNS name pointer chains in response packets. Enforces recursion depth limit and maximum name length.

## Tasks

- Implement `decompressName()` that reads a DNS name from a buffer at a given offset, following pointer chains
- Enforce a recursion depth limit of 10 to prevent malicious pointer loops
- Enforce maximum name length of 255 bytes
- Handle names at end of packet without compression
- Write unit tests (TDD) per the **Testing Strategy > Unit Tests > name-compression.ts** section:
  - Decompress pointers at various packet offsets
  - Handle chained pointers (pointer -> name containing another pointer)
  - Detect and reject pointer loops (recursion depth > 10)
  - Handle names at end of packet without compression

## Acceptance Criteria

- Pure function with no I/O or side effects
- All unit tests pass
- Pointer loops are detected and rejected gracefully (no infinite recursion)
