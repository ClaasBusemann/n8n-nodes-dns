Implement the DNS Lookup action node in `src/nodes/DnsLookup/DnsLookup.node.ts`.

## Context

From the design document, section **DNS Lookup Node (Action)**:

> The node's `displayName` is **DNS Lookup**. The internal class name is `DnsLookup`, and the node `name` is `dnsLookup`. Because the node has a single purpose, it does not use the `resource`/`operation` pattern.

Parameters from the design document:

| Parameter | Type | Description |
|-----------|------|-------------|
| Domain | String (expression-enabled) | The domain name to query |
| Record Type | Options | A, AAAA, MX, TXT, SRV, PTR, CAA, SOA, NS, CNAME, NAPTR, DNSKEY, TLSA |
| Resolver Mode | Options | Well-Known, Custom, Authoritative (auto-discover), System |
| Well-Known Resolvers | Multi-select | Cloudflare, Google, Quad9, OpenDNS, AdGuard, Control D |
| Custom Servers | Collection | Repeatable IP/port pairs |
| Timeout | Number | Default: 5000 |
| Retry Count | Number | Default: 1 |
| Recursion Desired | Boolean | Default: true |
| Output Consistency Check | Boolean | Default: true when querying multiple servers |

> Record Type and Resolver Mode set `noDataExpression: true`.

The runtime execution flow from the design document dispatches queries per input item via the DNS client, parses responses, and computes consistency flags for multi-server queries.

From section **n8n Linter Compliance**:

> Static subtitle: `subtitle: '={{$parameter["recordType"] + " lookup"}}'`

## Tasks

- Implement the `DnsLookup` node class extending `INodeType`
- Define all parameters from the table above with correct types, defaults, and display conditions
- Implement `execute()` following the runtime flow in the design document:
  1. Loop over input items
  2. Read parameters per item
  3. Resolve target servers based on resolver mode
  4. Dispatch queries via `Promise.allSettled()` using the DNS client
  5. Parse responses (decode packet, parse RDATA, parse TXT if applicable)
  6. Wrap results in `INodeExecutionData[]`
- Node class contains no packet encoding, no parsing, no UDP logic — delegates to transport and parsing modules
- Set `noDataExpression: true` on Record Type and Resolver Mode

## Acceptance Criteria

- Node appears in n8n with displayName "DNS Lookup"
- All parameters render correctly with proper display conditions
- Single-server queries produce the output format from the **Single Server Response** section
- Expression-enabled Domain parameter resolves per input item
- Node delegates all protocol logic to transport/parsing modules
