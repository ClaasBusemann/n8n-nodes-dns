Implement DNS packet encoding and decoding in `src/transport/dns-packet.ts`.

## Context

From the design document, section **Transport Layer > Wire Format**:

> The DNS wire protocol is defined in RFC 1035 §4. Each message consists of a fixed 12-byte header followed by variable-length question, answer, authority, and additional sections.

> The encoder builds a query packet from a domain name, record type, and flags. The decoder parses a response buffer into a structured object. Both operate entirely on `Buffer` using `readUInt16BE`, `writeUInt16BE`, and slice operations.

Key implementation details from the design document:
- **Transaction ID** — 2 random bytes via `crypto.randomBytes(2)`
- **Label encoding** — domain names encoded as length-prefixed labels terminated by a zero byte
- **RDATA parsing** — dispatched by TYPE field via a type map

From section **File Responsibilities**:

> **`dns-packet.ts`** — Pure functions. Encodes a query config object into a `Buffer` and decodes a response `Buffer` into a structured object. No I/O, no side effects.

## Tasks

- Implement `encodeQuery()` that takes a domain, record type, and flags and returns a `Buffer`
- Implement `decodeResponse()` that takes a response `Buffer` and returns a structured object with header, question, answer, authority, and additional sections
- Implement label encoding for domain names (length-prefixed labels + zero terminator)
- Use `crypto.randomBytes(2)` for transaction IDs
- Integrate name compression from KNX-00002 for decoding
- Write unit tests (TDD) per the **Testing Strategy > Unit Tests > dns-packet.ts** section:
  - Encode a query for every supported record type; assert exact byte sequences against reference packets
  - Decode reference response packets (static hex buffers) and assert correct parsing of all sections
  - Round-trip: encode -> decode -> compare to original input
  - Edge cases: maximum-length labels (63 bytes), maximum-length domain names (253 bytes), empty question section, zero-answer responses

## Acceptance Criteria

- Pure functions with no I/O or side effects
- Supports the 12-byte header layout shown in the design document
- All unit tests pass including edge cases
- Reference packets validated against real-world captures from `dig`
