Implement authoritative nameserver discovery via NS delegation chain walking in `src/utils/authoritative-discovery.ts`.

## Context

From the design document, section **Transport Layer > Authoritative Discovery**:

> When the user selects "Authoritative (auto-discover)" resolver mode, the node walks the DNS delegation chain:
> 1. Query any recursive resolver for the NS records of the target domain.
> 2. Extract nameserver hostnames from the answer section.
> 3. Resolve each nameserver hostname to an IP address (A record query).
> 4. Return the list of IPs as query targets.
>
> If the NS response includes glue records in the additional section (A records for the nameserver hostnames), the second resolution step is skipped.

From section **File Responsibilities**:

> **`authoritative-discovery.ts`** — Uses the DNS client to walk the NS delegation chain for a domain, returning the authoritative nameserver IPs. Optimizes by using glue records from the additional section when available.

## Tasks

- Implement `walkDelegationChain()` that takes a domain and returns authoritative nameserver IPs
- Query NS records for the target domain via a recursive resolver
- Extract nameserver hostnames from the answer section
- Resolve nameserver hostnames to IP addresses via A record queries
- Optimize: skip A queries when glue records are present in the additional section
- Write unit tests (TDD) per the **Testing Strategy > Unit Tests > authoritative-discovery.ts** section:
  - Mock DNS client returns staged NS and A responses
  - Verify correct two-step delegation chain walk
  - Verify glue record optimization (skip A query when additional section has IPs)
  - Handle failed NS lookups gracefully

## Acceptance Criteria

- Returns a list of authoritative nameserver IPs for a given domain
- Glue record optimization avoids unnecessary A queries
- Failed NS lookups are handled gracefully without crashing
- All unit tests pass with mocked DNS client
