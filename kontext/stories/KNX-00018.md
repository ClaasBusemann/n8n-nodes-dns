Create the GitHub Actions workflow for npm publishing with provenance attestation.

## Context

From the design document, section **Scaffolding and Tooling**:

> The project will include a GitHub Actions workflow (`.github/workflows/publish.yml`) for publishing to npm with provenance attestation, which becomes mandatory for verified community nodes on May 1, 2026.

From section **Publishing and Verification**:

> Publication uses the GitHub Actions workflow from the n8n starter template, producing an npm provenance attestation. The npm package author and the GitHub repository owner will match — a verification requirement.

From section **Test Infrastructure**:

> CI pipeline: `just test` on every commit (unit + integration); `just test-all` on PR merge
> Pre-publish: `just check` — runs lint + format-check + test-coverage + build + verify

## Tasks

- Create `.github/workflows/publish.yml` based on the n8n starter template
- Configure npm provenance attestation in the workflow
- Ensure npm package author and GitHub repository owner match
- Add CI checks: `just test` on every commit, `just test-all` on PR merge
- Include `just check` as a pre-publish gate (lint + format-check + test-coverage + build + verify)

## Acceptance Criteria

- Workflow publishes to npm with provenance attestation
- CI runs appropriate test suites per trigger (commit vs. PR merge)
- Pre-publish gate runs the full `just check` recipe
- Package author matches repository owner
