Implement multi-server output format with consistency and propagation flags in the DNS Lookup node.

## Context

From the design document, section **Output Format > Multi-Server Response**:

> When querying multiple servers, the output is one combined item per input item, containing a `results` array and top-level consistency flags.

```json
{
  "domain": "example.com",
  "type": "A",
  "results": [
    { "server": "1.1.1.1", "serverName": "Cloudflare", "answers": [], "responseTimeMs": 12 },
    { "server": "8.8.8.8", "serverName": "Google", "answers": [], "responseTimeMs": 18 }
  ],
  "consistent": true,
  "propagatedToAll": true,
  "serverCount": 3,
  "uniqueAnswers": 1
}
```

> `consistent` is `true` when all servers return identical answer values (ignoring TTL differences). `propagatedToAll` is `true` when all non-authoritative answers match the authoritative answer. These flags are designed to feed directly into downstream IF nodes.

## Tasks

- Implement `computeConsistencyFlags()` that compares responses across servers
- `consistent`: true when all servers return identical answer values (ignoring TTL)
- `propagatedToAll`: true when all non-authoritative answers match the authoritative answer
- Output `serverCount` and `uniqueAnswers` counts
- Structure multi-server output as one combined item per input item with a `results` array
- Integrate with the Output Consistency Check parameter (default: true when multi-server)

## Acceptance Criteria

- Multi-server output matches the structure in the design document
- Consistency comparison ignores TTL differences
- Propagation flag correctly compares against authoritative answer
- Single-server queries are unaffected (no `results` array wrapper)
